# v0.9 NCBI Release Candidate Kickoff

v0.9 is the release-polish lane. The goal is to turn a technically strong assembly and annotation into a submission package that is consistent across FASTA, AGP, GFF3, metadata, accession tracking, and reviewer-facing methods text.

This lane is intentionally conservative. A package is not ready because one validator passed or because the portal accepted an upload. It is ready when the files, metadata, and decisions all describe the same frozen release object.

## Scope

v0.9 focuses on:

- assembly and annotation submission order
- accession readiness across BioProject, BioSample, and SRA
- package-level decisions for assembly-only versus annotated submissions
- local audit helpers for release-package consistency
- submitter-facing handoff notes for table2asn review and discrepancy cleanup
- manuscript and reviewer-response language that matches what was actually submitted

## Release Principles

1. Freeze sequence names before annotation submission.
2. Keep one accession-tracking table for the whole project.
3. Treat `table2asn` validation and discrepancy outputs as a curation queue, not as an afterthought.
4. Do not submit a chromosome- or scaffold-level package without AGP when AGP is needed to describe the assembly.
5. Do not submit an annotated genome until BioProject, BioSample, and locus-tag planning are settled.

## Official Guidance To Check

These v0.9 pages align with current official guidance checked during drafting:

- NCBI genome submission portal: [About Genome (WGS) Submission](https://submit.ncbi.nlm.nih.gov/about/genome/)
- NCBI GFF3/annotation submission: [Annotating Genomes with GFF3 or GTF files](https://www.ncbi.nlm.nih.gov/genbank/genomes_gff/)
- NCBI SRA BioProject/BioSample path: [Submit your project and biological samples](https://www.ncbi.nlm.nih.gov/sra/docs/submitbio/)
- NCBI eukaryotic submission overview: [Submitting Eukaryotic Genome Data](https://www.ncbi.nlm.nih.gov/genbank/eukaryotic_submission)

Inference from those sources: for crop plant annotated submissions, the safest working order is still BioProject -> BioSample -> SRA/read plan -> frozen assembly package -> annotation package.

## Validation Target

The release lane should pass both of these checks:

```bash
scripts/audit_release_submission_decisions.py \
  examples/release_submission_decisions.tsv \
  -o /tmp/release_submission_decision_audit.tsv

scripts/check_release_bundle.py \
  --fasta 15_release/sample.genome.fa \
  --agp 15_release/sample.agp \
  --manifest examples/release_manifest.tsv \
  --out-dir 15_release/validation
```

## Release Rule

The package chosen for submission must be the same one described by the accession table, the release manifest, the methods text, and the annotation validation outputs.
