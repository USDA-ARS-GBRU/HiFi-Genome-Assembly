# v0.4 Release Candidate Checklist

Use this checklist before tagging the v0.4 curation module.

## Documentation

- `docs/v0.4_curation_index.md` links the workflow in the correct order.
- Step 8 in `README.md` points readers to the v0.4 curation workflow.
- Dotplot, PAF, IGV, RagTag, manual correction, rejected correction, and post-correction validation docs are present.
- The protocol clearly says automated tools are candidate generators, not automatic editors.
- Rejected corrections are documented as valid review outcomes.

## Helper Scripts

- `scripts/validate_breaks.py` validates breakpoint TSVs.
- `scripts/split_fasta_at_breaks.py` splits FASTA with a reproducible coordinate convention.
- `scripts/summarize_corrections.py` summarizes accepted edits.
- `scripts/audit_correction_decisions.py` audits accepted and rejected decision rows.
- `scripts/compare_fasta_stats.py` compares pre/post correction FASTA statistics.
- `scripts/make_correction_report.py` generates a Markdown report.

## sbatch Templates

- `01_sbatch_templates/minimap_assembly_paf.sbatch`
- `01_sbatch_templates/minimap_reference_to_assembly_igv.sbatch`
- `01_sbatch_templates/ragtag_correct_scaffold.sbatch`
- `01_sbatch_templates/post_correction_validation.sbatch`

## Examples

- Toy accepted break exists.

- Toy correction report can be generated end to end.

## CI and Local Validation

Before tagging:

```bash
python3 -m py_compile scripts/*.py
bash -n 01_sbatch_templates/*.sbatch
scripts/validate_breaks.py --fasta examples/toy/toy_assembly.fa --breaks examples/toy/toy_breaks.tsv -o /tmp/toy_breaks_validation.tsv
scripts/split_fasta_at_breaks.py --fasta examples/toy/toy_assembly.fa --breaks examples/toy/toy_breaks.tsv -o /tmp/toy_split.fa --map /tmp/toy_split.map.tsv
scripts/summarize_corrections.py --split-map /tmp/toy_split.map.tsv --decision-log examples/toy/toy_correction_decisions.tsv -o /tmp/toy_correction_summary.tsv --markdown /tmp/toy_correction_summary.md
scripts/audit_correction_decisions.py examples/toy/toy_correction_decisions.tsv -o /tmp/toy_correction_decision_audit.tsv
scripts/compare_fasta_stats.py --before examples/toy/toy_assembly.fa --after /tmp/toy_split.fa -o /tmp/toy_fasta_comparison.tsv
scripts/make_correction_report.py --sample toy --version 0.4.0-dev --decision-log examples/toy/toy_correction_decisions.tsv --decision-audit /tmp/toy_correction_decision_audit.tsv --correction-summary /tmp/toy_correction_summary.tsv --split-map /tmp/toy_split.map.tsv --fasta-comparison /tmp/toy_fasta_comparison.tsv --break-validation /tmp/toy_breaks_validation.tsv -o /tmp/toy_post_correction_report.md
```

## Tagging Rule

Tag v0.4 only when the curation workflow can be followed from suspicion to report without relying on undocumented manual steps.
