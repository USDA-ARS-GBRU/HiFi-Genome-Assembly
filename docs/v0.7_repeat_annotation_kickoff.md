# v0.7 Repeat Annotation Kickoff

v0.7 focuses on repeat annotation refinement for crop plant genome assemblies. Repeat annotation affects gene prediction, genome browser tracks, repeat landscapes, pan-genome comparisons, structural variant interpretation, and claims about genome size or transposable element expansion.

v0.7 can draft while v0.5 remains in human review and v0.6 continues T2T/completeness refinement. The main goal is to move from "run a masking tool" to a documented repeat library decision that can survive manuscript review.

## Goals

- Refine EDTA and RepeatModeler2/RepeatMasker workflows for crop genomes.
- Define how to choose a release repeat library or masking set.
- Track repeat library provenance and tool versions.
- Create repeat annotation decision logs for accepted and rejected masking runs.
- Standardize deliverables for soft-masked FASTA, repeat GFF3, repeat libraries, repeat summaries, and browser tracks.
- Connect repeat annotation decisions to downstream gene annotation.

## First Decisions

Before repeat annotation, decide:

- Is the FASTA sequence set stable enough to annotate?
- Are organellar and contaminant contigs removed or clearly separated?
- Is this the primary assembly, alternate haplotype, phased haplotype, or pan-genome representative?
- Will the project use EDTA, RepeatModeler2/RepeatMasker, a curated library, or a combined library?
- Which masked FASTA becomes the gene annotation input?
- How will repeat annotations be regenerated if chromosome names or sequence structure change?

## Review Standard

Repeat annotation should be considered release-ready only when:

- the masked fraction is plausible for the crop lineage
- the unclassified fraction is reviewed
- major TE classes are biologically plausible
- repeat outputs use final release sequence IDs
- soft-masked FASTA is available for gene annotation
- the repeat library has clear provenance
- downstream gene annotation uses the intended masked FASTA
