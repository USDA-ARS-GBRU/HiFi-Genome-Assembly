# v0.4 Curation Workflow Index

This is the recommended order for the v0.4 dotplot and misassembly curation workflow.

## 1. Triage Structure

Read:

- `docs/dotplot_misassembly_curation.md`
- `docs/paf_dotplot_options.md`
- `docs/dotplot_figures.md`

Generate whole-genome and focused dotplots. Use MUMmer and/or minimap2 PAF workflows.

## 2. Review Candidate Regions

Read:

- `docs/manual_correction_workflow.md`
- `docs/igv_breakpoint_reporting.md`
- `docs/minimum_evidence_checklist.md`

Use dotplots to choose questionable contigs. Map the reference genome to the HiFi assembly and inspect only those regions in IGV.

## 3. Record Decisions

Read:

- `docs/correction_decision_log_template.md`
- `docs/rejected_corrections.md`

Record accepted and rejected candidate edits. Rejected corrections are important project evidence.

## 4. Validate and Apply Breaks

Run:

```bash
scripts/validate_breaks.py --fasta original.fa --breaks sample.breaks.tsv -o sample.breaks.validation.tsv
scripts/split_fasta_at_breaks.py --fasta original.fa --breaks sample.breaks.tsv -o corrected.fa --map sample.split_map.tsv
```

## 5. Summarize and Audit

Run:

```bash
scripts/summarize_corrections.py --split-map sample.split_map.tsv --decision-log sample.correction_decisions.tsv -o sample.correction_summary.tsv --markdown sample.correction_summary.md
scripts/audit_correction_decisions.py sample.correction_decisions.tsv -o sample.correction_decision_audit.tsv
```

## 6. Validate Corrected Assembly

Read:

- `docs/post_correction_validation.md`
- `docs/agp_after_splitting.md`

Run the post-correction validation template when appropriate:

```bash
sbatch \
  --export corrected_fasta=corrected.fa,sample=sample,original_fasta=original.fa,breaks=sample.breaks.tsv,split_map=sample.split_map.tsv,decision_log=sample.correction_decisions.tsv \
  01_sbatch/post_correction_validation.sbatch
```

## 7. Report

Read:

- `docs/post_correction_report_template.md`
- `docs/correction_report_examples.md`

Generate a draft report:

```bash
scripts/make_correction_report.py \
  --sample sample \
  --decision-log sample.correction_decisions.tsv \
  --decision-audit sample.correction_decision_audit.tsv \
  --correction-summary sample.correction_summary.tsv \
  --split-map sample.split_map.tsv \
  --break-validation sample.breaks.validation.tsv \
  -o sample.post_correction_report.md
```
