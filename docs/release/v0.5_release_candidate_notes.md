# v0.5 Release Candidate Notes

These notes are a draft for a future `v0.5.0` tag. Do not tag `v0.5.0` until the remaining human review gates in `docs/release/v0.5_review_checklist.md` are resolved.

## Proposed Release Title

`v0.5.0: Scaffolding, finishing, and release-readiness review`

## Proposed Summary

This release candidate expands the PacBio HiFi crop plant genome assembly protocol beyond contig assembly into evidence-based chromosome-scale scaffolding, conservative gap filling, T2T-readiness triage, and public release preparation. It keeps the longform README as the current canonical protocol while building focused documentation pages that will support a future GitHub documentation site.

## Highlights

- Adds YaHS, 3D-DNA/Juicebox/JBAT, and RagTag scaffolding guidance.
- Frames RagTag and other reference-guided methods as comparison tools unless independent evidence supports accepted changes.
- Adds AGP definition, validation, and summary guidance.
- Adds conservative gap-filling workflow notes for LR_Gapcloser, TGS-GapCloser2, gapless, and TRFill.
- Adds telomere, centromere, gap, and T2T-readiness reporting helpers.
- Adds a worked scaffolding decision case that chooses evidence over maximum N50.
- Adds focused docs pages for assembly, QC, scaffolding, annotation, and release topics.
- Adds public release metadata checks, GitHub issue templates, and a pull request template.
- Adds a README-to-docs migration plan for the eventual web documentation split.

## Validation Included

The repository currently validates:

- Python helper syntax
- sbatch shell syntax
- README inventory consistency
- Markdown links
- README workflow coverage by focused docs
- required project metadata
- public release metadata consistency
- toy FASTA, AGP, correction, gap-filling, telomere, T2T, annotation, release-manifest, and QC dashboard workflows

## Known Limitations

- The repository is still in `0.5.0-dev`; stable release tagging requires final citation/contributor review.
- The longform README is intentionally large and has not yet been shortened into a final landing page.
- The beginner usability review template exists, but at least one outside-reader review should be completed before a stable milestone tag.
- Tool citations and version recommendations should continue to be checked against current upstream documentation during future releases.

## Proposed GitHub Release Text

```text
v0.5.0 expands HiFi-Genome-Assembly into a chromosome-scale scaffolding and release-readiness protocol for crop plant PacBio HiFi assemblies. It adds YaHS, 3D-DNA/Juicebox/JBAT, RagTag comparison guidance, conservative gap-filling notes, AGP review, T2T-readiness triage, focused documentation pages, public metadata checks, and a worked scaffolding decision case.

This is not a promise that every assembly should be scaffolded or corrected automatically. The central v0.5 principle is evidence-first review: contact maps, dotplots, read support, AGP validity, contamination screening, and biological plausibility should outweigh N50 alone.
```

## Tagging Dry Run

When the team is ready to tag, make one final release commit that:

1. Changes `VERSION` from `0.5.0-dev` to `0.5.0`.
2. Changes `CITATION.cff` version to `0.5.0`.
3. Updates `CITATION.cff` `date-released` to the tag date.
4. Changes README current roadmap version to `v0.5.0`.
5. Moves `CHANGELOG.md` heading from `0.5.0-dev` to `0.5.0`.
6. Updates `docs/status.md` so v0.5 is marked complete or maintained, and v0.6 becomes active development if appropriate.
7. Re-runs all validation checks.
8. Creates an annotated tag:

```bash
git tag -a v0.5.0 -m "v0.5.0: scaffolding, finishing, and release-readiness review"
git push origin v0.5.0
```

Only run the tag commands after the citation/contributor and beginner usability gates are complete.
