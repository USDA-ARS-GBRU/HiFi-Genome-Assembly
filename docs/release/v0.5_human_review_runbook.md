# v0.5 Human Review Runbook

This page records the human review packet that supported the v0.5 signoff and can still be reused as a compact stable-baseline review packet.

## Goal

Provide a compact review packet for the stable `v0.5.0` baseline.

Before handing this packet to an outside reviewer, run `v0.5_maintainer_precheck.md`.

## Review Packet

Ask the reviewer to use these pages:

1. [How to use this protocol in a real project](../how_to_use_this_protocol.md)
2. [Scaffolding worked decision case](../scaffolding_worked_decision_case.md)
3. [Release package decision guide](../release_package_decision_guide.md)
4. [v0.5 tagging readiness](v0.5_tagging_readiness.md)
5. [Beginner usability review](beginner_usability_review.md)

## Reviewer Prompts

The reviewer should answer:

- can a beginning graduate student find the right entry page without help?
- does the scaffolding logic feel conservative and evidence-based?
- is the release path understandable without needing prior NCBI experience?
- are there any places where the docs overclaim stability or completeness?
- would you trust this repo as the starting point for a real project in the lab?

## Minimum Recording Requirements

Record:

- reviewer name
- review date
- pages reviewed
- major confusion points
- whether the issues are wording, navigation, or workflow contradictions
- overall recommendation: `tag v0.5.0` or `keep v0.5.0-dev`

## Decision Rule Used For Signoff

`v0.5.0` was signed off when:

- the reviewer finds only small wording or navigation issues
- no major contradictions are found between README, docs, examples, and helper scripts
- maintainers agree that the live docs site is now the correct public entry point

The signoff would have been delayed if:

- a reviewer still gets lost in the intended reading path
- the release logic is still too hard for a new user to follow
- active-draft lanes create confusion about what is stable

## Suggested Continuing Use

After any future stable-baseline review, update:

- `examples/beginner_usability_review.tsv`
- `docs/status.md`
- `CHANGELOG.md`, if substantial readability fixes are made
