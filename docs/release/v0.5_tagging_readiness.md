# v0.5 Tagging Readiness

This page is the short decision point for whether the repository should move from `v0.5.0-dev` to a stable `v0.5.0` tag.

## Current Position

`v0.5.0-dev` is now strong on content breadth, docs structure, helper-script coverage, and public documentation deployment. The remaining gap is mostly editorial and release-governance, not missing workflow machinery.

## What Looks Ready

- scaffolding and finishing guidance has a worked decision case
- gap filling and T2T-readiness guidance exist and are connected
- repeat and gene annotation have first-pass draft baselines
- NCBI release and bundle validation paths are documented
- the docs site is live and the root README is a proper landing page
- helper audits and toy fixtures are present across major workflow areas

## What Still Separates `v0.5.0-dev` From `v0.5.0`

- one outside-reader beginner-usability pass should still be completed and recorded
- final citation and contributor wording should be reviewed one more time
- the project should decide whether any remaining active-draft wording in v0.6-v0.9 creates confusion for a user who expects only stable content

## Tagging Recommendation

Tag `v0.5.0` when all of the following are true:

- the outside-reader pass produces only small wording or navigation fixes
- no major workflow contradictions are found between README, docs, helper scripts, and examples
- maintainers are comfortable presenting the current docs site publicly as the recommended entry point

Keep `v0.5.0-dev` if:

- a reviewer still finds major ambiguity in the scaffolding or release logic
- the team wants one more readability pass for new graduate students
- there is uncertainty about how stable the current wording should be for public citation

## Recommended Immediate Next Checks

1. run or record one beginner-usability review
2. review `CITATION.cff` and contributor wording
3. decide whether `v0.5.0` should mean "stable scaffolding-and-release baseline" while `v0.6-v0.9` remain active-draft lanes

For a concrete sequence of those steps, use `v0.5_human_review_runbook.md`.
