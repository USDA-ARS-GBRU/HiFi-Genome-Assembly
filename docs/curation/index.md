# Curation

This section focuses on evidence-first structural review and conservative manual correction.

## Read This Section First If

- a dotplot suggests a chimeric contig or scaffold
- reference alignments break in suspicious places
- automated correction tools are proposing more breaks than you trust
- you need methods language for why a correction was made or rejected

- [v0.4 curation workflow index](../v0.4_curation_index.md)
- [Dotplot and misassembly curation](../dotplot_misassembly_curation.md)

- [IGV breakpoint reporting](../igv_breakpoint_reporting.md)

## Practical Outcome

After this section, you should have:

- a documented rationale for each accepted or rejected break
- a corrected FASTA only where evidence is strong
- post-correction validation tables and report text
- a cleaner assembly for scaffolding or release
