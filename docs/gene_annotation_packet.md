# Gene Annotation Packet

This packet is the shortest ordered route through the v0.8 gene-annotation material.

## Packet Goal

Use this packet to choose a release gene set that balances evidence support, biological plausibility, ID continuity, and release readiness.

## Recommended Packet Order

1. [v0.8 gene kickoff](v0.8_gene_annotation_kickoff.md)
2. [Gene annotation strategy](gene_annotation.md)

## What This Packet Should Produce

By the end of this sequence, you should have:

- a release-gene-set decision
- a clear evidence path for functional annotation
- an annotation QC summary that is ready for internal review or release planning
- a worked example showing how to resolve disagreement between competing gene-set candidates
- a worked example for balancing familiar inherited IDs against supported novel biology

## Before You Move On

Before leaving this packet, you should be able to answer:

- Why is the chosen release gene set stronger than the comparison candidates?
- Which loci were retained for biological novelty and which were rejected as weak models?
- How will downstream users understand ID continuity, changed models, and new loci?

## When To Use This Page

Use this packet when:

- v0.8 is the current working lane
- you want a compact internal review order for gene-set decisions
- a new contributor needs a small but complete annotation entry path
