# v0.8 Gene Annotation Kickoff

v0.8 focuses on gene annotation refinement for crop plant genome assemblies. The goal is to move from "run an annotation tool" to a documented release gene-set decision that can survive manuscript review, NCBI validation, and downstream reuse.

v0.8 can draft while v0.5 remains in human review and while v0.6/v0.7 continue refinement. The main objective is to connect repeat masking, evidence selection, structural gene prediction, QC, and release filtering into one reproducible decision path.

## Goals

- Refine Liftoff, BRAKER3, and MAKER workflows for crop genomes.
- Define how to choose a release gene set across transfer, de novo, and hybrid evidence strategies.
- Track evidence provenance for RNA-seq, Iso-Seq, proteins, and lifted annotations.
- Standardize QC outputs for BUSCO protein mode, gene counts, transcript counts, protein sets, and TE-derived prediction review.
- Connect gene annotation decisions to repeat masking, table2asn review, and release bundles.

## First Decisions

Before annotation, decide:

- Which soft-masked FASTA is the annotation input?
- Which evidence types are available: RNA-seq, Iso-Seq, proteins, reference annotation?
- Is the project primarily transfer-based, de novo, or hybrid?
- Which annotation run will be treated as comparison-only versus release-track?
- How will TE-derived models, short fragments, and unsupported loci be filtered?
- Will the release gene set preserve reference IDs or use new IDs?

## Review Standard

Gene annotation should be considered release-ready only when:

- the genome and repeat mask inputs are frozen
- evidence sources are documented
- the release gene set is traceable to exact inputs and commands
- BUSCO protein-mode and basic structural QC are reviewed
- TE-derived and low-confidence models are handled transparently
- the final GFF3 and FASTA products match release sequence IDs
- table2asn validation is reviewed when annotation submission is planned
