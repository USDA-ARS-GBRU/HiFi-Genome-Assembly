SV Calling per ChatGPT - needs validation


Below is a practical, “works in plant genomes” approach with two tiers:

* **Tier A (recommended, rich classification):** **SyRI + MUMmer4** (excellent for inversions/translocations/duplications)
* **Tier B (quick, lightweight):** **minimap2 + paftools** (fast SV list, less rich)

# Step 02A (recommended): SyRI SV calls from MUMmer4 alignments

## Inputs

* `ref.fa` = your backbone assembly (FASTA; ideally chromosome-level, but contig-level is fine)
* `qry.fa` = another RIL assembly

### 0) Pre-clean: remove tiny contigs and standardize

This avoids tons of spurious “rearrangements”.

* Filter out contigs < 10 kb (or 50 kb, depending on assembly fragmentation)
* Make sure sequences are A/C/G/T only (N’s fine)

Example (seqkit):

```bash
seqkit seq -m 10000 ref.fa > ref.min10k.fa
seqkit seq -m 10000 qry.fa > qry.min10k.fa
```

---

## 1) Whole-genome alignment with nucmer (MUMmer4)

```bash
nucmer --maxmatch -l 100 -c 500 ref.min10k.fa qry.min10k.fa -p ref_vs_qry
delta-filter -1 ref_vs_qry.delta > ref_vs_qry.1delta
show-coords -THrd ref_vs_qry.1delta > ref_vs_qry.coords
```

Notes:

* `--maxmatch` is common for SV/rearrangement discovery, but can be noisy in repeats. If repeat noise is high, switch to `--mum` or tighten `-l/-c`.
* `delta-filter -1` keeps “best 1-to-1” chains; often improves SyRI stability.

---

## 2) Run SyRI to call rearrangements + local SV

```bash
syri -c ref_vs_qry.coords \
     -r ref.min10k.fa \
     -q qry.min10k.fa \
     -F B \
     --prefix ref_vs_qry
```

Outputs you care about:

* `ref_vs_qrysyri.out` (main event table)
* `ref_vs_qrySV.txt` (insertions/deletions/duplications etc., depending on version/options)
* `ref_vs_qry.log` (always read if something looks weird)

---

## 3) Convert SyRI calls to VCF (optional but useful)

SyRI typically provides a helper:

```bash
syri --makevcf -r ref.min10k.fa -q qry.min10k.fa -s ref_vs_qrysyri.out -o ref_vs_qry.syri.vcf
```

(Exact flag names vary a bit by SyRI version; if your install differs, I’ll adapt quickly once you paste `syri -h` output.)

---

## 4) Repeat for all 3 non-backbone assemblies

You’ll end up with 3 SyRI callsets:

* `ref vs RIL2`
* `ref vs RIL3`
* `ref vs RIL4`

---

## 5) Merge SV callsets into a nonredundant “4-genome catalog”

SyRI is pairwise, so merging matters.

Typical approach:

* Convert each to VCF (or TSV with type/start/end)
* Merge using **SURVIVOR** (good at merging SV across samples with breakpoint fuzz)

Example (if you have VCFs):

```bash
ls *.syri.vcf > vcf_list.txt
SURVIVOR merge vcf_list.txt 1000 1 1 -1 -1 -1 merged.sv.vcf
```

Interpretation of parameters (common setting):

* `1000` = merge breakpoints within 1 kb
* `1` = require same SV type
* `1` = require same strand/orientation (depends on your use)
  You can tune this once you see breakpoint noise.

---

## 6) Produce the breeder-friendly “SV summary tables”

From the merged SV catalog, create:

* counts by type and size bin
* “high-impact SVs” list:

  * inversions > 1 Mb
  * deletions overlapping genes
  * duplications overlapping resistance clusters

And for inversions specifically, create a table like:

* `chr  start  end  size  present_in  breakpoint_repeat_context  genes_inside`

That table becomes your “introgression barrier candidates” list.

# Step 02B (fast alternative): minimap2 + paftools

If you just want a first-pass SV list quickly:

## 1) Align assemblies with minimap2

```bash
minimap2 -x asm5 -t 24 --cs ref.min10k.fa qry.min10k.fa > ref_vs_qry.paf
```

## 2) Call variants with paftools (gives indels and some SV-ish events)

```bash
paftools.js call -f ref.min10k.fa ref_vs_qry.paf > ref_vs_qry.paftools.vcf
```

This is fast and useful for large indels, but it’s **not as strong as SyRI** for complex rearrangements/inversions.
