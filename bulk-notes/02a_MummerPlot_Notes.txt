Good question — those symbols actually carry **real information**, not just decoration, and they’re easy to misinterpret if you haven’t stared at a lot of MUMmer plots.

I’ll break this into **(A) what the circles mean**, **(B) why they’re spaced the way they are**, and **(C) why you sometimes see purple + light blue vs only purple**.

---

## A) What the circles along the MUMmer plot lines indicate

Each **circle is an individual alignment block** (technically a maximal exact match–derived alignment segment) produced by **nucmer** and then rendered by **mummerplot**.

So:

* The **line** = the visual interpolation of collinear alignment blocks
* The **circles** = the **actual discrete alignments** used to draw that line

Each circle corresponds to:

* one high-confidence aligned region between reference and query
* typically hundreds bp to many kb long (depending on parameters and genome complexity)

If you removed the circles, you’d still see the diagonal line — but you’d lose information about:

* alignment density
* fragmentation
* repeat-induced breaks

### Key takeaway

> **Circles = anchor points of real alignments.
> Lines = visual connection between those anchors.**

---

## B) Why the circles cluster, spread, or vary in density

This tells you a lot about genome structure and assembly quality.

### 1) Dense clusters of circles

Usually mean:

* repetitive regions
* segmental duplications
* gene family expansions
* transposon-rich areas
* assembly fragmentation in one or both genomes

In plant genomes, you’ll often see clusters near:

* pericentromeres
* resistance gene clusters
* recent tandem duplications

### 2) Evenly spaced circles

Usually indicate:

* clean, unique, collinear sequence
* good assembly contiguity
* low repeat interference

That long smooth diagonal you’re seeing is a **very good sign**.

### 3) Gaps between circles

May indicate:

* small insertions/deletions
* divergent sequence
* masked repeats
* contig boundaries (very common)

If gaps line up across many comparisons, they’re often:

* real biological indels
* conserved structural features

---

## C) Why some plots have **purple + light blue**, and others only **purple**

This part depends on **strand orientation**.

### Default MUMmer color convention

* **Purple (or red)**
  → alignments in the **same orientation** (forward strand)
  → positive slope diagonal

* **Light blue (or green)**
  → alignments in the **reverse orientation**
  → negative slope diagonal

So:

* **Both purple and light blue present**
  → you have **inversions** or inverted segments

* **Only purple present**
  → everything is collinear and forward-oriented

---

## When you *should* worry

Here’s when circles/colors become diagnostic:

### 🚩 Alternating purple and light blue blocks

→ likely **large inversion(s)**

### 🚩 Dense crisscross of small circles

→ repetitive collapse or over-alignment

### 🚩 Broken diagonal into multiple fragments

→ misassembly, translocation, or contig join error

### 🚩 Blue-only or purple-only on a chromosome

→ orientation mismatch (entire contig reversed)

This dotplot is your **visual sanity check** before trusting SyRI or paftools.

Rule of thumb:

* **Dotplot looks clean → SV calls are trustworthy**
* **Dotplot looks chaotic → fix assembly / alignment params first**

For inversions in particular:

* you’ll see a **clean light-blue diagonal segment**

## One last practical tip (highly recommended)

When generating mummer plots for SV work, always:

* turn **gridlines on**
* label axes with contig/chromosome IDs
* generate one plot per chromosome if possible

That way, when you see:

> “Inversion on chr05: 18.2–26.7 Mb”

You can **visually confirm it in 5 seconds**.
